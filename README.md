# ZipComic2XTC
压缩包格式漫画转XTC/XTCH格式工具

理论支持阅星瞳X3 X4（无机器测试，在另一个软件上测试正常）
需要安装Python运行环境，会自动安装剩余所需的运行组件，需下载UnRAR.exe（跟py文件放一起）才能运行
或者使用命令打包成单文件：

控制台版本：python -m PyInstaller --onefile --console --name zipcomic2xtc --add-data "UnRAR.exe;." --collect-all numba --collect-all llvmlite --hidden-import numpy --hidden-import py7zr --hidden-import rarfile zipcomic2xtc.py

GUI：python -m PyInstaller --onefile --noconsole --name zipcomic2xtc_GUI --add-data "UnRAR.exe;." --collect-all numba --collect-all llvmlite --hidden-import numpy --hidden-import py7zr --hidden-import rarfile zipcomic2xtc_GUI.py

支持转换的格式：
XTC (1-bit 黑白，容器格式，单文件)
XTCH (2-bit 4级灰度，容器格式，单文件)
XTG (1-bit 黑白，单页模式，每个图片单独输出)
XTH (2-bit 4级灰度，单页模式，每个图片单独输出)

可选择转换分辨率：
X4（480×800）（默认）
X4双倍分辨率（960×1600）
X3 （528×792）
X3双倍分辨率（1056×1584）
自定义分辨率

漫画白边自动裁切：
开（默认）/关

画面切割：
横切2图：4种模式，支持滚动切图
横切3图：4种模式，支持滚动切图

漫画内横版图片自动旋转：
不旋转
顺时针旋转90度（默认，x4按键在右面）
逆时针90度

拉伸至全屏：
开（默认）/关

抖动强度：
默认0.7（可调范围0-1）

容器格式分包：
4G（FAT32），自定义大小或不分包

同时处理的线程数量：
默认为CPU线程数，不建议修改，经测试R9 5900HX默认就能跑满CPU

项目需求：漫画压缩包转 Xteink 设备格式（含图形界面）

1. 总体目标
开发一个 Python 程序，用于将常见漫画压缩包（如 ZIP、CBZ、7Z、CBR、RAR）中的图片转换为 Xteink 设备（X4/X3 等分辨率）支持的 XTC、XTCH、XTG、XTH 格式。程序同时提供控制台交互版本（zipcomic2xtc.py）和图形化界面版本（zipcomic2xtc_GUI.py），二者共享核心处理逻辑（core.py）。所有用户提示及界面文字均使用中文。

2. 功能模块划分

2.1 核心处理模块（core.py）
- 使用 Numba 加速的 Floyd‑Steinberg 抖动算法，支持 1‑bit（黑白）和 2‑bit（4 级灰度）输出，抖动强度可调。
- 图像预处理：透明背景填充为白色，可选黑白边自动裁切，横版图片整体旋转（仅对未切割的图片生效）。
- 切割逻辑：
  - 直接切割：将竖版图片按给定比例垂直分割为 2 或 3 部分。
  - 滚动切割：根据重叠比例计算 3 或 5 个部分的垂直范围，相邻部分保留重叠。
- 缩放：拉伸至目标分辨率或按比例缩放并居中填充黑边。
- 编码：根据格式生成 XTG（1‑bit）或 XTH（2‑bit）二进制数据，容器格式（XTC/XTCH）的组装在调用层完成。
- 多进程支持：每张图片（或 GIF 每帧）的处理在独立进程中完成，进程数由用户指定。

2.2 控制台版本（zipcomic2xtc.py）
- 启动后依次询问所有转换参数，并提供默认值。
- 支持以下选项：
  - 输出格式（XTC/XTCH/XTG/XTH）
  - 目标分辨率（预设 X4/X3 及双倍，或自定义宽高）
  - 自动裁切黑白边（是/否）
  - 横版图片旋转方式（顺时针/逆时针/不旋转）
  - 画面切割模式（不切割/横切2图/横切3图）及其详细子选项（比例、滚动重叠比例）
  - 拉伸至全屏（是/否）
  - 抖动强度（0~1）
  - 并发进程数
  - 单页模式下的文件名格式（编号/漫画名-编号）
  - 容器模式下的分包大小（4G/自定义/不分包）
- 递归遍历用户指定的目录，查找所有支持格式的压缩包。
- 使用 ProcessPoolExecutor 并行处理每个压缩包内的图片，实时显示进度。
- 日志记录：在程序目录下创建 log 文件夹，每次运行生成带时间戳的日志文件，记录所有操作及错误信息。
- 解压 RAR 文件时，优先使用程序根目录下的 UnRAR.exe（若存在），否则回退到 rarfile 库。
- 若检测到缺失的依赖（Pillow、py7zr、rarfile、natsort、numpy、numba），自动通过 pip 安装。
- GIF 动图处理：若压缩包中包含 GIF，询问用户选择“只处理第一帧”或“处理所有帧”，整个运行过程仅询问一次。

2.3 图形界面版本（zipcomic2xtc_GUI.py）
- 基于 tkinter 构建，主窗口布局：
  - 顶部：目录设置（输入目录、输出目录，带浏览按钮）
  - 中间：水平分割窗（ttk.PanedWindow）
    - 左侧：参数选项区，采用双列布局，内容较多时支持垂直滚动
    - 右侧：日志显示区，使用 ScrolledText 自带滚动条
  - 底部：转换进度（总进度条、当前文件进度条）及控制按钮（开始转换、停止、关于）
- 所有参数通过图形控件（单选按钮、复选框、输入框、下拉框等）设置，实时反映动态选项（如分辨率自定义、切割子选项、单页/容器模式下的特殊选项）。
- 提供“关于”窗口，显示软件名称、版本号及 GitHub 链接（可点击打开浏览器）。
- 窗口可调整大小，并设置最小尺寸（900×700）以保证顶部和底部控件始终可见。当窗口宽度缩小时，右侧日志区域优先被压缩（甚至隐藏），左侧参数区保持基本可见。
- 启动时自动检查并安装缺失依赖（复用控制台版本的依赖检查函数）。
- 转换过程异步执行（线程），通过回调更新进度条和日志区域，支持停止操作（需等待当前图片处理完成）。
- 日志同时写入文件（与控制台版本共用日志目录）并显示在图形界面日志区。

3. 图像处理流程（控制台与 GUI 一致）
1. 预处理：透明背景填充白色 → 可选黑白边裁切 → 转换为灰度。
2. 整体旋转：若图片为横版且用户选择了旋转方式，则整体旋转（旋转后不再参与切割）。
3. 切割（仅竖版、非首尾页且用户选择切割时执行）：
   - 直接切割：按比例分割成 2 或 3 块。
   - 滚动切割：按重叠比例计算相邻块重叠区域，输出 3 或 5 块。
   - 切割后的每个子图应用用户选择的旋转方式（顺时针/逆时针/不旋转）。
4. 缩放：拉伸至目标分辨率，或按比例缩放后居中填充黑边。
5. 抖动与编码：使用 Numba 加速的 Floyd‑Steinberg 抖动，生成 1‑bit 或 2‑bit 数据，然后按格式编码为 XTG/XTH 二进制。

4. 输出与文件管理
- 容器格式（XTC/XTCH）：
  - 将所有图片数据合并为一个容器文件，按用户选择的分包大小拆分为多个文件（如 漫画名.xtc、漫画名-1.xtc）。
  - 容器头部包含元数据（标题、作者、时间戳）、索引表等。
- 单页模式（XTG/XTH）：
  - 在输出目录下创建以压缩包名命名的文件夹，内部按用户指定的文件名格式存放单页文件（1.xtg 或 漫画名-1.xtg）。
- 文件名中的非法字符（<>:"/\|?*）自动移除，并在日志中记录。

5. 依赖与部署
- 所需第三方库：Pillow、py7zr、rarfile、natsort、numpy、numba。
- 程序启动时自动检测并安装缺失依赖（通过 pip）。
- 支持通过 PyInstaller 打包为单文件可执行程序，需将 UnRAR.exe 嵌入，并在代码中正确处理路径（sys._MEIPASS）。
- 日志目录自动创建，日志文件按时间命名，便于排障。

6. 使用示例（控制台）
1. 运行 zipcomic2xtc.py。
2. 按提示选择格式、分辨率、裁切、旋转、切割、拉伸、抖动强度、并发数等。
3. 输入包含压缩包的目录路径和输出目录。
4. 程序自动处理所有压缩包，显示进度并输出日志。
5. 处理完成后按回车退出。
